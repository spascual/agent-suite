# Documentation Standards

## Naming

- Name functions/classes/variables with clear preconditions

## Docstrings

- Only if function/class/variable behavior unclear from just name, write one-sentence docstrings (no input/output) otherwise no docstrings

## Reference Tools

- For up-to-date specific python library docs, use context7 MCP
